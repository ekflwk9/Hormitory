## GitHub CommitConvention   
### 협업 중 사용한 커밋 컨벤션을 정리합니다.      
*   **type:** 커밋의 종류를 나타냅니다.   
    *   `Add`:	새 파일 추가      
    *   `Remove`:	기존 파일 삭제     
    *   `Feat`:	새로운 기능   
    *   `Fix`:	기능 수정   
    *   `Refactor`:	기능 변경 없이 코드 구조만 개선   
    *   `Merge`:	브랜치 머지 작업   
    *   `Docs`:	README 등 문서 작업   
    *   `Test`:	테스트 코드 추가   
    *   `Style`:	들여쓰기, 세미콜론 등 스타일만 바꿈   
    *   `Chore`:	빌드, 패키지 설정 등 기타 사항    
    *   `Design`:	UI 디자인 변경
